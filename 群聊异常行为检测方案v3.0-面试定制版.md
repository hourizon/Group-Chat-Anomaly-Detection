## 二、核心检测算法 (C++ 实现)

### 1. 内容安全：朴素贝叶斯文本分类器

- **目标**: 替代简单的关键词匹配，让程序通过学习样本自主判断消息的风险概率。
- **C++ 实现步骤**:
    1.  **训练模块 (`Trainer`)**: 
        - 读取带标签的语料库（如 `spam.txt`, `ham.txt`）。
        - 对文本进行分词（可基于空格或简单标点）。
        - 统计每个词在“垃圾”和“正常”类别中出现的次数，计算**条件概率 P(词|类别)**。
        - 将计算出的概率模型（词频、类别先验概率等）序列化并保存到模型文件（如 `bayes_model.dat`）。
    2.  **分类模块 (`Classifier`)**:
        - 启动时加载 `bayes_model.dat`。
        - 接收到新消息时，对其分词。
        - 应用**贝叶斯公式**，结合模型中的概率，计算该消息属于“垃圾”类别的**后验概率**。
        - 若概率超过预设阈值（如 0.95），则判定为风险内容。

### 2. 行为安全：统计画像分析器

- **目标**: 替代固定的硬编码阈值，通过统计学方法动态发现异常行为。
- **C++ 实现步骤**:
    1.  **基线分析 (离线或启动时)**: (可选，也可简化为硬编码) 分析大量历史正常日志，计算出“正常用户”的行为基线，如“平均每小时加群数”的标准差和均值。
    2.  **实时画像 (`Profiler`)**: 
        - 在内存中（使用 `std::map` 和 `std::deque`）实时维护每个用户近期的行为序列（如加群时间戳）。
        - 动态计算用户的短期行为特征（如过去1小时加群数）。
    3.  **异常判断 (`Detector`)**: 
        - 将用户的实时行为特征与行为基线进行比较。
        - 当一个用户的行为显著偏离正常基线（如超过3个标准差），或触及预设的“绝对红线”（如10分钟内加5个群），则判定为异常。

## 四、实施方案

### Phase 1: 朴素贝叶斯分类器 C++ 实现
- **目标**: 用 C++ 从零构建一个完整的文本分类器。
- **任务**:
    1.  设计用于存储词频和概率的数据结构。
    2.  实现 `Trainer` 类，完成从文本文件学习并生成模型文件的功能。
    3.  实现 `Classifier` 类，完成加载模型并预测新文本概率的功能。
    4.  编写 `main` 函数，串联训练和分类过程，进行单元测试。

### Phase 2: 统计画像分析器 C++ 实现
- **目标**: 实现基于滑动窗口的行为检测逻辑。
- **任务**:
    1.  实现 `BehaviorProfiler` 类，用 `std::map` 和 `std::deque` 维护用户行为时间序列。
    2.  实现滑动窗口清理机制，移除过期事件。
    3.  实现异常判断逻辑。

### Phase 3: 框架集成与主程序开发
- **目标**: 将算法模块与 QQ Bot 框架连接起来。
- **任务**:
    1.  搭建 `go-cqhttp` 环境。
    2.  在 C++ 主程序中，实现一个 Websocket/HTTP 客户端，用于接收 `go-cqhttp` 的事件推送。
    3.  编写事件分发逻辑，将 `群消息` 送入贝叶斯分类器，将 `群成员增加` 等事件送入画像分析器。
    4.  实现调用 `go-cqhttp` 的 API 来执行踢人、禁言等处置操作。
